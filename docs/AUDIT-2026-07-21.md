# Architectural & Code-Quality Audit — `yt-tui`

**Date:** 2026-07-21
**Auditor role:** Principal Go Architect / Systems Quality Engineer
**Scope:** all hand-written code under `internal/` (generated `*.pb.go` / `*connect.go` excluded)
**Verification:** `go build` ✅ · `go vet` ✅ · `go test -race ./...` ✅ (all green) · `golangci-lint` 3 issues · `gofmt -l` **33 files non-compliant**

---

## 1. Executive Architectural Summary

This is a **well-architected codebase in the middle of a disciplined refactor**, and it shows in both directions. The layer model is explicit and mechanically enforced: `.golangci.yml` depguard rules pin down foundation → domain → service → api → tui flow, the `api.Backend` interface cleanly separates the TUI from InProc/Remote transports (a genuinely good design for the upcoming Phase-5 daemon split), and the service layer (`FeedService`, `ChannelService`) declares its ports (`FeedRepo`, `RecommendSource`) where they are consumed — textbook idiomatic Go. The MVU discipline is largely intact: side effects go through `tea.Cmd`, messages are typed, and `Root.Update` is a clean dispatcher. The new `videotable` abstraction (`ColumnDef[T]`, capability interfaces like `HasDuration`, `EnrichAll`) is a strong consolidation of what was previously per-tab table code.

The weaknesses are concentrated in three places. First, **the refactor is unfinished and the seams show**: `search.go` and the channel-list pane of `channels.go` still hand-roll the navigation logic that `TableNav` was built to eliminate (~250 duplicated lines), `table_helpers.go` carries transitional dead code, and 33 files fail `gofmt`. Second, **testing is inverted relative to risk**: the pure domain packages (feed filtering, channels, media, library) are well tested with table-driven tests, but the TUI layer (~7,000 lines, including every `Update` state machine), the db layer (~2,200 lines), and both `api.Backend` implementations have **zero tests**. Third, there are **two real concurrency defects** — a download-event pipeline that can permanently block download goroutines once its sole consumer disappears, and a playback-position tracking design that can attribute one video's position to another and leaks an untracked goroutine per playback.

Nothing here is rotten; the trajectory is clearly positive (the git history shows systematic consolidation). The priority order is: fix the two concurrency defects, finish the TableNav/column-factory migration already started, then put a test harness around the `Update` loops and db layer before the Phase-5 remote transport multiplies the surface area.

### 🔴 Critical

#### F-1 — Download event pipeline can permanently block download goroutines

* **Location:** `internal/downloader/downloader.go:173-176, 230, 296` + `internal/api/inproc.go:437-475` + `internal/tui/tab/downloading.go:214-236`
* **Anti-Pattern:** Blocking channel sends with a single, cancelable consumer; unbounded producer set.
* **Impact:** `run()` and `fail()` send to `d.eventCh` (buffer 64) unconditionally. The only drain path is the `InProc.Events` bridge goroutine, which exits when its `ctx` is canceled **without anything continuing to drain `d.eventCh`**. After that (or if the Downloading tab never subscribes, or if `Events` is called twice — the bridge steals from a single shared channel), every active download goroutine blocks forever on its next progress line: goroutine leak, semaphore slots never released, download system wedges silently. A long video emits hundreds of progress events, so the 64-slot buffer is minutes of protection at best.
* **Recommendation:** make sends non-blocking (progress events are droppable), or move to a subscriber registry with per-subscriber buffered channels.

```go
// Before (downloader.go:173)
d.eventCh <- Event{Kind: EventProgress, VideoID: item.Video.ID, ...}

// After — progress is lossy by nature; never let UI slowness stall a download
func (d *Downloader) emit(ev Event) {
    select {
    case d.eventCh <- ev:
    default: // drop oldest-style: drain one, then push
        select { case <-d.eventCh: default: }
        select { case d.eventCh <- ev: default: }
    }
}
```

For completion/error events (must not be lost), pair the drop-guard with the `dlItemsMsg` polling that already exists (`fetchItemsCmd` re-reads authoritative state, so a dropped event is self-healing).

#### F-2 — Playback-position tracking: cross-playback data race + goroutine-per-playback leak

* **Location:** `internal/tui/app/root.go:449-467` (ticker goroutine), `root.go:474-482` (`playerWaitCmd`), `internal/device/player/mpris.go:46-76, 131-152`
* **Anti-Pattern:** Untracked goroutine with no cancellation; shared mutable player identity across overlapping playbacks; `sync.Once` reset (`b.once = sync.Once{}` at `mpris.go:66`) racing with a concurrent `Close()` from the previous process-exit goroutine.
* **Impact:**
  1. `playCmd(A)` spawns a ticker goroutine bound to video **A**'s ID but reading `r.player.Position()` — a **global** current-position. Start video **B** while A's player is exiting and A's ticker/final-save writes **B's position under A's ID**. Resume points silently corrupt.
  2. The old exit-goroutine's `b.Close()` can fire the **new** `sync.Once` (reset in `exec` at mpris.go:66) and close the **new** `stopCh`, killing position polling for the just-started playback.
  3. Each playback leaks a ticker goroutine until process exit if `Wait()` never returns (e.g. detached `Setsid` player adopted by init).
* **Recommendation:** Make a playback a first-class value — `exec` returns a `*Session{videoID, doneCh, position()}`; the ticker and `playerWaitCmd` capture the session, not the backend. Delete the `sync.Once` reset; use a generation counter or per-session stop channel.

```go
// Before (root.go): ticker bound to id, position read from shared player
go func() { ...; if p, _ := r.player.Position(); p > 0 { save(id, p) } ... }()

// After: session owns identity + lifecycle; no cross-talk, ctx-cancelable
sess, err := r.player.Start(src.URI, title, pos, audioOnly)
return r, watchSessionCmd(r.backend, id, sess) // reads sess.Position(), waits sess.Done()
```

#### F-3 — ~70 % of the codebase (all stateful layers) has zero tests

* **Location:** `internal/tui/**` (no `_test.go` at all), `internal/db/**`, `internal/api/**`, `internal/backend/**`
* **Anti-Pattern:** Coverage inversion — pure functions tested, state machines untested.
* **Impact:** Every `Update()` handler (tab switching, chord/goto state, sort chords, pane transitions, edit modes), all SQL, and both Backend implementations are verified only by manual use. The in-flight table migration is exactly the kind of change that silently breaks `PageDown` on one tab out of ten.
* **Recommendation:** Three cheap, high-leverage harnesses — all table-driven, matching the existing style:
  1. **`videotable.TableNav`** — pure, zero deps, an afternoon of tests covering gg/G/`:n`/paging/circular edge cases (empty list, single row, numBuf restore at `nav.go:103-105`).
  2. **db** — `New(t.TempDir(), …)` per test; migrations already run; cover history/positions/playlists round-trips and the versioned migrations.
  3. **Tab Update loops** — a `fakeBackend` struct implementing `api.Backend` (one mock, reused everywhere) + message-driven tests: `tab.Update(chsLoadedMsg{...})` → assert rows/state. No TTY needed; models are plain values.

```go
func TestTableNavPageDown(t *testing.T) {
    tests := []struct{ name string; rows, start, height, want int; circular bool }{
        {"clamps at bottom", 10, 8, 5, 9, false},
        {"empty list", 0, 0, 5, 0, false},
        // ...
    }
    for _, tc := range tests { t.Run(tc.name, func(t *testing.T) { /* ... */ }) }
}
```

#### F-4 — Navigation logic duplicated 4× despite `TableNav` existing

* **Location:** `internal/tui/tab/search.go:452-631` (three near-identical blocks for `chTable`, `vidTable`, `drillTable`), `internal/tui/tab/channels.go:334-350` (pane-0 Up/Down), plus transitional helpers in `table_helpers.go:21-56`

#### F-5 — `api.Backend` is a ~100-method god-interface

* **Location:** `internal/api/client.go:65-166`
* **Anti-Pattern:** Interface Segregation violation; "header interface" exported from producer instead of consumer-declared contracts.
* **Impact:** Every tab receives all 100 methods but uses 3–8. Mocking for tests (F-3) means stubbing 100 methods. Adding any db method forces `InProc`, `Remote`, and the Connect transport to change in lockstep — friction that Phase 5 will multiply. It also erases the read/write and local/network distinction the daemon split will need.
* **Recommendation:** Keep `Backend` as the transport-level union (it must be one wire contract), but have consumers declare narrow views — the codebase already does this correctly in `service.FeedRepo`. Tabs accept their own port:

```go
// tab/history.go — consumer-declared, trivially fake-able
type historyBackend interface {
    HistoryVideos(ctx context.Context, limit int) ([]domain.HistoryEntry, error)
    VideoHistory(ctx context.Context, videoID string) ([]domain.HistoryEntry, error)
    HasLocalVideo(ctx context.Context, id string) (domain.LocalVideo, bool)
    DeleteLocalVideo(ctx context.Context, id string) error
    // ...
}
func NewHistory(backend historyBackend, ...) History
```

`*InProc` / `*Remote` satisfy these automatically; nothing else changes.

#### F-6 — Presentation concerns leaked into the domain layer

* **Location:** `internal/domain/history.go:39-57` (`GetIndicator` returns `" ● "`, `GetLabel` returns `"Streamed"/"Downloaded"`), `internal/domain/local.go:44-52`
* **Anti-Pattern:** Domain → presentation coupling (the depguard rules block the *import*, but the glyphs and display labels moved in anyway to satisfy `videotable` interfaces).
* **Impact:** Terminal glyphs and English display strings are now business-layer API; a daemon returning `HistoryEntry` over the wire ships UI bullets. Changing the indicator style means editing `domain`.
* **Recommendation:** The wrapper-row pattern already used by `HistoryRow`/`ChannelRow` is the right home. Move `GetIndicator`/`GetLabel`/`GetBaseTitle` implementations onto the tab-layer row types (or a `videotable` adapter), leaving `domain` with data + semantic predicates only:

```go
// domain: semantic fact
func (e HistoryEntry) IsDownloadEvent() bool { return strings.HasPrefix(e.EventType, "download") }

// tab layer: presentation choice
func (r HistoryRow) GetIndicator() string {
    if r.IsAudio() || r.IsDownloadEvent() { return " ● " }
    return " ○ "
}
```

#### F-7 — History delete performs file deletion inside the tab, swallowing every error

* **Location:** `internal/tui/tab/history.go:265-276`
* **Anti-Pattern:** Presentation layer doing storage side effects; four consecutive swallowed errors on a destructive operation.
* **Impact:** `os.Remove` on user media from a `tea.Cmd`; if removal fails the DB row is deleted anyway (orphaned file), and the user sees "Deleted: …" regardless. In remote mode this is also simply wrong — the file lives on the daemon host.
* **Recommendation:** Push the composite operation behind the backend (`DeleteVideoCompletely(ctx, id) error` — db rows + file, transactional-ish) and surface the error:

```go
// Before (tab layer)
_ = os.Remove(lv.FilePath)
_ = t.backend.DeleteLocalVideo(ctx, lv.ID)
_ = t.backend.DeleteVideoHistory(ctx, e.VideoID)

// After
if err := t.backend.DeleteVideoCompletely(ctx, e.VideoID); err != nil {
    return tuipkg.StatusMsg{Text: "delete: " + err.Error(), IsErr: true}
}
```

#### F-9 — `downloader` imports Bubble Tea; dead `WaitForEvent`

* **Location:** `internal/downloader/downloader.go:18, 347-351`
* **Anti-Pattern:** Infrastructure → presentation-framework dependency; dead code (no callers found).
* **Impact:** The download engine can't be compiled into the Phase-5 daemon without dragging in a TUI framework; conceptually inverted layering that depguard doesn't currently police.
* **Recommendation:** Delete `WaitForEvent` and the `tea` import; add a depguard rule denying `charm.land/bubbletea` outside `internal/tui`.

#### F-10 — `Search` struct: 25 fields, 4 interleaved modes (SRP)

* **Location:** `internal/tui/tab/search.go:38-75`
* **Anti-Pattern:** God-object; the input/recent-history/results/drill-down state machines share one flat field set (`recentMode`, `histIdx`, `recentCursor`, `recentVS`, `onVideos`, `drillCh`, …) with implicit invariants (e.g. `histIdx` must reset whenever `recentMode` flips — currently maintained by hand at 6 call sites).
* **Impact:** Highest-defect-risk file in the TUI; hard to test even after F-3.
* **Recommendation:** Extract a `recentSearches` sub-model (its own `Update`/`View`/cursor) and a `drill` sub-model; `Search` becomes a router across `inputMode | recentMode | resultsMode | drillMode`. The recent-searches list is also the last hand-rolled list view using `nav.Move` — folding it into a table/`TableNav` removes the `internal/tui/nav` package's last consumer.

#### F-11 — Fire-and-forget error swallowing at decision points

* **Location:** 63 sites; notable: `backend/service/feed.go:58` (`_ = s.repo.SaveFeedCache` — cache silently stale), `root.go:447` (`_ = r.backend.AddHistory`), `root.go:456` (`SaveVideoPosition` in ticker), `videotable/auxdata.go:25-27` (three ignored errors → tab silently renders no positions/watched state)
* **Anti-Pattern:** Swallowed errors without even a debug log; the project has an `internal/debug` package that these could use.
* **Recommendation:** Rule of thumb: user-invisible periodic writes may drop errors **with a debug log**; anything that changes what the user sees (`LoadAuxDataCmd`) should surface a `StatusMsg` on first failure.

#### F-12 — Magic strings for overlay kinds and event types

* **Location:** `tui.go:78` + `root.go:288-297` (`"video_detail"`, `"add_to_playlist"`), `domain/history.go:64-66` (`"streamAudio"`, `"download audio"`, `"playAudio"` — note the inconsistent space vs camelCase), `root.go:137/143` (`"stream"`, `"play"` prefixes concatenated with `"Video"/"Audio"` at `root.go:443-447`)
* **Impact:** The event-type vocabulary is assembled by string concatenation in one file and parsed by `strings.HasPrefix` in another; a typo compiles fine and silently breaks the audio indicator and history labels.
* **Recommendation:** `type OverlayKind int` + constants; a `domain.EventType` string type with named constants and constructor `EventType(action Action, audio bool)`.

#### F-13 — Mutable package-level globals in `render`

* **Location:** `internal/tui/render/render.go:17-21, 42, 68`
* **Anti-Pattern:** Package-level mutable state (`ColDuration`, `ColDurationPos`, `ColDate`, `activeDurFmt`, `activeDateFmt`) written by `SetDurFmt`/`SetDateFmt` and read from every render path.
* **Impact:** Column factories capture widths at construction time (`w := render.ColDurationPos` in `DurationCol`), so a `SetDurFmt` after any table exists desyncs header vs cells; also a data race if ever called post-startup. Currently safe only by convention ("call once at startup").
* **Recommendation:** Either make them genuinely immutable (compute in `main` before UI construction, pass a `render.Formats` value into the theme/styles), or at minimum document the invariant on the vars and guard with a `sync.OnceFunc` panic on second call.

#### F-14 — `sortedChannels()` re-sorts on every keypress and every View

* **Location:** `internal/tui/tab/channels.go:479-521` (called from `View`, `handleKey`, `handleKeyFlat`, `handleEditInput` — up to 3× per event)
* **Impact:** O(n log n) + full slice copy per keystroke; more importantly, cursor index ↔ sorted-order coupling is fragile — if `chLatest` changes between the sort in `handleKey` and the sort in `View`, the highlighted row silently points at a different channel (the unsubscribe path acts on `sorted[idx]`).
* **Recommendation:** Cache the sorted slice as a field, invalidated on `chsLoadedMsg`/sort-mode change/`subs` mutation — same pattern the table rows already follow.

#### F-15 — Width/alignment helpers: byte-length centering + duplicate `ralign`

* **Location:** `videotable/columns.go:187-199` (`calign` uses `len(s)`; `ralign` duplicated), `tab/table_helpers.go:15-17` (unused twin, flagged by lint), `render.go:164-181` uses `runewidth` correctly
* **Impact:** Any non-ASCII column header (e.g. a future "Größe" or "時間") mis-centers; two helpers with one dead.
* **Recommendation:** Delete `tab.ralign`; base `calign` on `runewidth.StringWidth`.
